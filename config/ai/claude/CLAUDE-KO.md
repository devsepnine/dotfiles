# 클로드 코드 구성

## 중요: 가이드 우선순위

**'/.guides/'는 클로드의 모든 기본 동작을 재정의합니다.**.

작업 전 필수로 무조건 확인을 진행해야한다:
1. '/guides/'에 관련 가이드가 있는지 확인합니다
2. 가이드를 정확히 읽고 따르세요
3. 가이드 규칙 > 클로드 기본값(항상)

## 코어 루프

문제 정의 → 작은 안전 변경 → 변경 검토 → 리팩토링 - 반복합니다.

## 필수 규칙

** 작업전 /guides/ 에 관련 가이드가 존재하는지 필수로 확인**

1. 변경하기 전에 관련 파일을 읽어보세요.
2. 업무, 커밋, 홍보를 작게 유지하세요.
3. 이슈/PR/ADR에 가정을 기록합니다.
4. 모든 입력을 검증하고 출력을 인코딩/정규화합니다.
5. 성급한 추상화를 피하고 의도를 드러내는 이름을 사용하세요.
6. 결정하기 전에 최소한 두 가지 대안을 비교하세요.

## 임계 한계

- 파일 ≤ 300 LOC
- 함수 ≤ 50 LOC
- 매개변수 ≤ 5
- 사이클로매틱 복잡도 ≤ 10

## 보안 규칙

**NEVER:** 코드, 로그 민감 데이터, SQL 주입/XSS/CSRF의 비밀

**항상:** 입력 검증, 매개변수화된 쿼리, 인증 확인
